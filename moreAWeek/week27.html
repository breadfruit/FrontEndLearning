    # Tips

    # 背景

    缺少本地调试，组件库因此浪费多个版本号tag。

    # 措施

    库使用接入的依赖。

    # 怎么做

    1. 在nbd-lib-mobile进行yarn link，这时候会在{prefix}/lib/node_modules/<package创建文件夹。

    其中prefix可以通过在终端进行 npm prefix -g查看

    1. 在使用方进行yarn link pakage-name进行映射。
    2. 删除软链接
    
npm unlink page-name

    # FAQ

    当使用方的依赖和库有相同的依赖，导致包重复。

    可以查看库的package.json中有这样一段

    peerDependencies在install的时候是不会去安装的。

    { 
	"scripts": {
			"build:dumi": "dumi build",
	},
	"peerDependencies": {
    "react": "^17.0.0",
    "react-router-dom": "^5.2.0"
  }
}

    查看yarn.lock文件，可以看umi在构建的时候会安装react安装包。所以在进行调试的时候肯定也是从构建产物的文件中读取。

    因此我们需要组件库去link接入方的依赖

    错误做法：

    yarn link /usr/local/lib/node_modules/fe-nbd-ykd/node_modules/react

    正确做法：

    yarn link ../../fe-nbd-ykd/node_modules/react

    查看lock文件指定包的依赖关系 npm ls pack-name
