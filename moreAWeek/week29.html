    # Tips

    除了 filter 还有什么置灰网站的方式？ (opens new window) CodePen (opens new window)

    filter作用于当前元素，并且它的后代元素也会继承这个属性

    mix-blend-mode作用于元素的直系父元素的内容和元素的背景

    backdrop-filter 作用于元素背后的所有元素

    # 全屏置灰

    filter

    html {
    filter: grayscale(.95);
    -webkit-filter: grayscale(.95);
}

    svg

    <svg xmlns="https://www.w3.org/2000/svg">
  <filter id="grayscale">
    <feColorMatrix type="matrix" values="0.3333 0.3333 0.3333 0 0 0.3333 0.3333 0.3333 0 0 0.3333 0.3333 0.3333 0 0 0 0 0 1 0"/>
    </filter>
</svg>

    html {
    filter: url(#grayscale);
}

    # 首屏置灰遮罩

    借助 pointer-events: none 保证页面交互

    特定的图形元素可以成为事件的目标。 none则指任务遮罩不作为事件目标。

    现象： 当然，这里有个很严重的问题，我们的页面是存在大量交互效果的，如果叠加了一层遮罩效果在其上，那这层遮罩下方的所有交互事件都将失效，譬如 hover、click 等。

    pointer-events: none;

    backdrop-filter

    兼容比较差

    html {
    position: relative;
    width: 100%;
    height: 100%;
    overflow: scroll;
}
html::before {

    position: absolute;
